১. প্রজেক্ট পরিচিতি ও ফাইল স্ট্রাকচার

PulseNetwork-এর আর্কিটেকচার অত্যন্ত সহজ এবং মডার্ন। এটি মূলত একটি Next.js (React) ফ্রন্টএন্ড এবং একটি Express.js (Node.js) ব্যাকএন্ড সার্ভারের সমন্বয়ে তৈরি। এর সবচেয়ে বড় সুবিধা হলো, প্রজেক্টের পুরো কোডটি শুধুমাত্র একটি প্রোগ্রামিং ভাষা—JavaScript ব্যবহার করে লেখা হয়েছে।

২. এক্সপ্রেস এপিআই রাউট ডকস (API Documentation)

ফ্রন্টএন্ড ক্লায়েন্ট থেকে ডেটা রিড ও রাইট করার জন্য ব্যাকএন্ডে নিম্নলিখিত RESTful API এন্ডপয়েন্টগুলো ব্যবহার করা হয়েছে:

POST /api/auth/register

রক্তদাতা, হাসপাতাল বা সাধারণ রোগী হিসেবে রেজিস্ট্রেশন করার জন্য ব্যবহৃত হয়।

Request Body: { name, email, password, role, phone, division, district, thana }
POST /api/auth/login

লগইন এপিআই, যা ইমেইল ও পাসওয়ার্ড যাচাই করে একটি সেশন টোকেন রিটার্ন করে।

Request Body: { email, password }
GET /api/donors

ভৌগোলিক সীমানা (বিভাগ/জেলা/থানা) এবং ব্লাড গ্রুপ ফিল্টার অনুযায়ী রক্তদাতাদের লিস্ট খোঁজার জন্য ব্যবহৃত হয়।

Query Params: ?division=Dhaka&district=Dhaka&blood_group=O%2B
GET /api/geography

বাংলাদেশের সকল বিভাগ, জেলা এবং থানার স্ট্রাকচার্ড ডাটা জেসন ফরমেটে লোড করে ফ্রন্টএন্ড সিলেক্টর ড্রপডাউনগুলো পপুলেট করতে ব্যবহৃত হয়।

৩. রিয়েল-টাইম চ্যাট ও ওয়েবসকেট লজিক

রক্তদাতার সাথে রোগীরা যেন তাৎক্ষণিকভাবে যোগাযোগ করতে পারেন, সেজন্য কোনো লেটেন্সি বা পেজ রিলোড ছাড়াই চ্যাট সিস্টেমটি সচল রাখতে WebSocket Protocol (ws) ব্যবহার করা হয়েছে।

💬 চ্যাট মেসেজ আদান-প্রদান লজিক:

  1. কানেকশন এস্টাব্লিশমেন্ট: ক্লায়েন্ট পেজ লোড হওয়ার সাথে সাথে তার সেশন টোকেন হেডার দিয়ে ws:// প্রোটোকলে ব্যাকএন্ড সার্ভারের সাথে একটি পারসিস্টেন্ট TCP কানেকশন খোলে।
  2. মেসেজ ডিসপ্যাচ: যখন কোনো ইউজার মেসেজ টাইপ করে সেন্ড করেন, তখন ফ্রন্টএন্ড থেকে একটি JSON স্ট্রিং `{ type: 'CHAT_MSG', receiver_id: 'xx', message: 'hello' }` সার্ভারে যায়।
  3. সার্ভার রাউটিং: সার্ভার মেসেজটি ফাইল ডাটাবেজে রেকর্ড করে এবং টার্গেট রিসিভার যদি অনলাইনে থাকেন, তবে তার ওপেন সকেটে মেসেজটি সাথে সাথে ফরওয়ার্ড করে দেয়।

৪. ইন্টারেক্টিভ ম্যাপ ও স্থানাঙ্ক কোঅর্ডিনেশন

ম্যাপের ক্ষেত্রে আমরা বাংলাদেশের প্রকৃত ভৌগোলিক বাউন্ডারি ডেটা ব্যবহার করেছি। কোনো থার্ড পার্টি ম্যাপ প্রোভাইডার ব্যবহার না করে, শুধুমাত্র হালকা ভেক্টর ইমেজে ম্যাপ রেন্ডার করার জন্য SVG ব্যবহার করা হয়েছে।

🗺️ কীভাবে মানচিত্রের কোড তৈরি করা হয়েছে?

বাংলাদেশের অফিসিয়াল জিওগ্রাফিক কোঅর্ডিনেট GeoJSON ফাইল থেকে ৮টি বিভাগের সীমানা পয়েন্টগুলো রিড করা হয়েছে। এরপর অক্ষাংশ এবং দ্রাঘিমাংশ ডাটাকে একটি কাস্টম প্রোজেকশন ম্যাথমেটিক্যাল ক্যালকুলেটরের মাধ্যমে SVG বাউন্ডিং বক্স সাইজ `0 0 437.8 601.2` এ কনভার্ট করে SVG path আকারে রেন্ডার করা হয়েছে।

৫. সোর্স কোড লাইব্রেরি ও ক্রেডিট (Third Party & Credits)

প্রজেক্টটি ডেভলপ করতে ওপেন সোর্স কমিউনিটির যে সকল কোড লাইব্রেরি ও সোর্স ব্যবহার করা হয়েছে, তা নিচে উল্লেখ করা হলো:

  • 🗺️ Bangladesh Division GeoJSON Coordinates সোর্স: GitHub Open-source Bangladesh Shapefile Repository। বাংলাদেশের ৮টি বিভাগের সর্বশেষ অরিজিনাল ম্যাপ ডাটা পয়েন্ট সংগ্রহ করতে এই সোর্সটি ব্যবহার করা হয়েছে।
  • ⚡ Next.js (React.js Framework) সোর্স: NPM Next Library। সিঙ্গেল পেজ অ্যাপ্লিকেশন রাউটিং, হট রিলোড এবং ডাইনামিক রেন্ডারিংয়ের জন্য রিয়্যাক্ট ফ্রেমওয়ার্ক ব্যবহার করা হয়েছে।
  • 🎨 Tailwind CSS (Utility Framework) সোর্স: Tailwind CSS CDN & PostCSS loader। ওয়েবসাইটের ক্লিন লাইট কালার থিম এবং মোবাইল রেসপন্সিভ গ্রিড ডিজাইন করতে ব্যবহৃত হয়েছে।
  • 💬 WebSocket Node.js (ws) সোর্স: NPM ws package। রিয়েল-টাইম কানেকশন এবং ইনস্ট্যান্ট মেসেজ রাউটিংয়ের লাইভ চ্যাট ব্যাকএন্ড হ্যান্ডেল করতে ব্যবহৃত হয়েছে।
